# 智能体（Agent）测试要点

> 基于 Seaf 平台已知功能整理 | 更新于 2026-04-17

---

## 一、测试范围

智能体模块涉及三个平台：构建侧（开发平台）、用户侧（用户平台）、管理侧（管理平台）。

---

## 二、功能测试要点

### 2.1 智能体基本信息

| 测试点 | 说明 |
|--------|------|
| 创建智能体 | 必填项校验、名称长度限制、图标上传 |
| 编辑智能体 | 修改后保存、数据一致性 |
| 删除智能体 | 软删除/硬删除、关联资源处理 |
| 空间隔离 | 跨空间资源不可见（权限矩阵必测） |

### 2.2 智能体详情页 4 个 Tab

#### 编排 Tab

- 工作流/编排配置保存与解析
- 节点配置渲染正确性

#### 日志 Tab（已完整）

| 测试点 | 说明 |
|--------|------|
| 日志列表加载 | 分页、排序、时间范围筛选 |
| 用户名称展示 | 正常用户名（`t_user.user_name`）/ 空值兜底 |
| 会话ID / 日志ID | 唯一性、链接跳转 |
| 「日志」入口 | 点击打开日志回放，内容完整性 |
| 「执行链」入口 | Langfuse 链接有效性 |

**用户名称为空场景**（需兜底）：
- API 调用
- 分享链接调用
- 用户侧调用

#### 发布 Tab

| 测试点 | 说明 |
|--------|------|
| 发布按钮权限 | 仅构建侧可见 |
| 发布范围选择 | 仅本空间 / 全部空间 |
| 免审核开关影响 | 管理侧配置变更后即时生效 |
| 审核状态流转 | 待审核→通过 / 拒绝→重新发布 |

#### 运营 Tab

- 数据统计准确性（待补充具体指标）

## 三、权限与角色测试（必须覆盖）

| 角色 | 构建侧 | 用户侧 | 管理侧 |
|------|--------|--------|--------|
| 开发者 | 全部功能 | - | - |
| 普通用户 | - | 对话/收藏/评价 | - |
| 管理员 | - | - | 审核/资源管理/数据统计 |

### 必测权限边界

- [ ] 跨空间越权访问（A 空间用户不能看到 B 空间智能体）
- [ ] 未登录用户访问限制
- [ ] API 调用的权限校验（Token 鉴权）
- [ ] 管理员误操作权限兜底

## 五、状态机覆盖（每个箭头至少一条用例）

```
未提交审核(template_check=-1)
    │ 用户点击发布
    ▼
待审核(template_check=1, publish_status=2)
    │
    ├── 通过(template_check=2, publish_status=2)
    │     │ 管理侧下架
    │     ▼
    │   未发布(template_check=2, publish_status=1)
    │     │ 用户重新发布
    │     ▼
    │   提交审核(回到待审核)
    │
    ├── 拒绝(template_check=3, publish_status=1)
    │     │ 用户重新发布
    │     ▼
    │   提交审核(回到待审核)
    │
    └── 撤销(template_check=4, publish_status=1)
          │ 用户重新发布
          ▼
        提交审核(回到待审核)
```
