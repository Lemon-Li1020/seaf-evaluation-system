## 一、Coze（扣子）

### 平台概况

- **开发商**：字节跳动
- **定位**：一站式 AI Bot 开发平台，无需代码快速搭建聊天机器人
- **核心模块**：Bot 编排 / 插件 / 知识库 / 工作流 / 多渠道发布

### 测试要点

| 模块 | 测试重点 |
|------|----------|
| **工作流编排** | 节点执行顺序、循环/判断逻辑、代码节点异常 |
| **插件调用** | API 参数传递、返回结果解析、超时处理 |
| **知识库** | 检索精度、切片策略、召回率 |
| **多渠道发布** | 各渠道 Bot 行为一致性 |
| **发布审核** | 审核状态流转、驳回重提逻辑 |

### Seaf 对比

- Coze 有丰富的插件生态市场，Seaf 暂无 → 插件丰富度是短板
- Coze 多渠道分发能力强 → Seaf 无此功能
- 工作流测试方法可参考 Coze 画布编排的节点测试

## 二、Dify

### 平台概况

- **开发商**：开源社区（Apache 2.0）
- **定位**：开源 LLM 应用开发平台，支持 AI 工作流
- **核心模块**：工作流编排 / RAG / 多模型 / API 开放

### 测试要点

| 模块 | 测试重点 |
|------|----------|
| **LLM 调用** | 模型切换、Prompt 模板变量替换、输出格式一致性 |
| **RAG 管道** | 文档解析、切块策略、向量检索、幻觉率 |
| **工作流** | 节点类型覆盖（LLM/Code/Tool/Condition/Loop） |
| **API 接口** | 契约测试、鉴权、限流、异常返回 |
| **日志与监控** | Token 消耗统计、调用链路 |

### Seaf 对比

- Dify 开源可控，支持私有化部署 → Seaf 目前无开源版本
- Dify RAG 能力成熟 → Seaf 知识库模块需对标
- Dify 日志监控（内置调用日志）→ Seaf 日志 Tab 已实现类似功能

## 四、行业通用 AI Agent 测试方法论

> 综合 IEEE / ISO AI 系统测试标准 + 大厂实践

### 4.1 功能正确性

| 测试点 | 方法 |
|--------|------|
| LLM 输出质量 | 人工评估 + 语义相似度 + 任务完成率 |
| Tool/Plugin 调用 | Mock 外部 API，验证调用参数和返回值 |
| 工作流节点 | 每个节点输入/输出边界测试 |
| 多轮对话记忆 | 上下文窗口内信息保持度验证 |

### 4.2 性能与稳定性

| 测试点 | 方法 |
|--------|------|
| 并发能力 | 阶梯压测（1→10→100→1000 并发） |
| 响应时间 P99 | 收集延迟分布，关注长尾 |
| Token 消耗 | 统计单次/累计 Token 用量 |
| 长时间运行 | 7×24h 稳定性监控 |

### 4.3 安全与伦理

| 测试点 | 方法 |
|--------|------|
| Prompt 注入 | 红队测试，注入恶意指令 |
| 越权访问 | 跨空间/跨角色资源访问 |
| 输出安全 | 敏感词过滤、脱敏验证 |
| 数据隐私 | 日志脱敏、用户数据隔离 |

### 4.4 可靠性

| 测试点 | 方法 |
|--------|------|
| 第三方服务降级 | Mock 外部依赖，验证降级策略 |
| 超时/断网 | 网络模拟（延迟/丢包/断连） |
| 数据一致性 | 幂等性验证，重复调用结果一致 |

### 4.5 Seaf 测试策略建议

| 维度 | Seaf 当前能力 | 建议补充 |
|------|--------------|----------|
| 功能测试 | ✅ 三平台基本功能 | 补充工作流编排测试 |
| 接口测试 | ✅ 有 API 文档 | 补充鉴权/越权用例 |
| 集成测试 | ⚠️ 联调中 | 补充跨平台联动测试 |
| 安全测试 | ⚠️ 待补充 | 补充权限矩阵测试 |
| 性能测试 | ⚠️ 待补充 | 补充并发/响应时间测试 |
