### 2.2 模块职责

| 模块 | 职责 | 核心能力 |
|------|------|----------|
| **评测核心服务** | 评测集管理、任务调度、结果汇总、回归检测 | 评测集 CRUD、任务状态管理、结果持久化 |
| **推理评测器** | 调用推理 Agent、采集日志、工具调用校验 | Agent 调用、日志解析、工具匹配 |
| **工作流评测器** | 触发工作流、采集节点性能、判定流程完整性 | 工作流执行、节点耗时采集、状态判定 |
| **LLM 评委服务** | 统一打分入口、Prompt 管理、模型切换 | 多维度打分、评分归一化、模型降级 |
| **报告服务** | 报告生成、版本对比、Grade 计算、告警 | 数据聚合、差异计算、通知触发 |

## 三、技术选型

### 3.1 技术栈

| 层级 | 技术选型 | 说明 |
|------|----------|------|
| **后端框架** | Python/FastAPI | 轻量、高性能、支持异步 |
| **数据库** | PostgreSQL + Redis | PostgreSQL 存储结构化数据；Redis 缓存任务状态 |
| **消息队列** | Celery + Redis | 评测任务异步执行，支持分布式 |
| **LLM 调用** | DeepSeek-V3 API | 成本低、速度快，支持结构化输出 |
| **配置管理** | YAML | 评测维度权重、Prompt 模板配置化 |
| **日志** | 结构化日志（JSON） | 方便日志分析和问题定位 |

### 3.2 外部接口依赖

| 依赖接口 | 调用方式 | 说明 |
|----------|----------|------|
| Seaf chat_agent_ask | HTTP REST | 推理 Agent 对话接口，支持流式返回 |
| Seaf workflow_execute | HTTP REST | 工作流执行接口 |
| DeepSeek-V3 API | HTTP REST | LLM 评委打分 |

  concurrency: 5

## 六、安全设计

### 6.1 权限控制

| 角色 | 权限 |
|------|------|
| 管理员 | 全部功能 |
| 评测员 | 创建评测任务、查看报告 |
| 查看者 | 仅查看报告 |

### 6.2 数据隔离

- 不同团队的评测数据通过 `team_id` 字段隔离
- 评测任务只允许操作本团队的智能体

### 6.3 敏感信息

- API Key 等敏感信息通过环境变量注入，不写入配置文件
- 评测结果中的敏感内容自动脱敏

## 七、监控与告警

### 7.1 关键指标

| 指标 | 说明 | 阈值 |
|------|------|------|
| 评测任务成功率 | 成功完成的评测任务 / 总任务 | < 95% 告警 |
| LLM 调用成功率 | LLM 评委调用成功 / 总调用 | < 99% 告警 |
| 评测任务耗时 | 单次评测任务平均耗时 | > 10min 告警 |
| 队列积压数 | 等待执行的评测任务数 | > 100 告警 |

### 7.2 日志规范
