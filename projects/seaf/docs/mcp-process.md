## 一、MCP / Skill 发布流程

> MCP 与 Skill 的发布、审核、下架流程**完全一致**，下同。

### 1.1 发布操作

用户在构建侧发起发布请求，发布时可以选择目标范围：

| 发布范围 | 配置条件 | 审核要求 |
|----------|----------|----------|
| **仅本空间** | 管理侧开启「仅本空间成员」免审核开关 | ✅ 开关打开 → 免审，直接发布<br>❌ 开关关闭 → 仍需审核 |
| **全部空间** | 无特殊条件 | ❌ 必须经管理侧审核通过，才能上线供用户使用 |

### 1.2 流程图

```
用户选择发布范围
     │
     ├── "仅本空间"
     │      │
     │      └── 管理侧「仅本空间成员免审核」开关
     │                 │
     │                 ├── ✅ 打开 → 直接发布上线（仅本空间可见）
     │                 └── ❌ 关闭 → 进入审核队列
     │
     └── "全部空间"
            │
            └── 必须进入管理侧审核 → 通过后上线（全空间可见）
```

### 1.3 结论

- **"全部空间"** → 必须审核，不存在免审路径
- **"仅本空间"** → 依赖管理侧开关，**开关打开才免审**，否则仍需审核

## 二、MCP / Skill 下架流程

### 2.1 操作路径

**管理侧**点击「下架」按钮

### 2.2 效果

| 位置 | 表现 |
|------|------|
| 构建侧列表 | 状态显示为「**未发布**」 |
| 详情页右上角 | 显示「**发布**」按钮 |

### 2.3 后续操作

用户点击「发布」按钮后，**重新走发布流程**（即回到本文档第一节的逻辑）。

---

## 三、MCP / Skill 审核不通过流程

### 3.1 操作路径

**管理侧**点击「**审核不通过**」

### 3.2 效果

| 位置 | 表现 |
|------|------|
| 构建侧列表 | 状态显示为「**审核不通过**」 |
| 详情页右上角 | 显示「**发布**」按钮 |

### 3.3 后续操作

用户点击「发布」按钮后，**重新走发布流程**。

---

## 四、MCP / Skill 审核中详情页展示规则

### 4.1 触发条件

当前状态为「**审核中**」时，用户打开详情页。

### 4.2 页面限制

审核中的详情页**仅显示以下内容**：

1. **基本信息**（名称、描述、图标等）
2. **工具列表**（MCP → tools；Skill → skill 工具）

**不支持**其他编辑或操作。

## 五、前后端交互逻辑

### 5.1 列表页

**接口**：`GET /api/mcp/list`（MCP）、`GET /api/skill/list`（Skill）

**返回字段说明**：

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | int | 唯一标识 |
| `user_id` | int | 创建者用户 ID |
| `user_name` | string | 创建者用户名 |
| `user_avatar` | string | 创建者头像 URL |
| `team_id` | int | 团队 ID |
| `name` | string | 名称 |
| `detail` | string | 描述 |
| `publish_status` | int | **发布状态**（见 5.2） |
| `images` | string | 图标 URL |
| `create_time` | string | 创建时间（ISO 8601） |
| `update_time` | string | 更新时间（ISO 8601） |
| `is_auth` | int | 认证状态 |
| `auth_type` | int | 认证类型 |
| `mcp_config` / `skill_config` | string | 配置（JSON 字符串） |
| `mcp_type` / `skill_type` | int | 类型 |
| `mcp_category` / `skill_category` | int | 分类 |
| `flow_id` | int | 关联流程 ID |
| `auth_desc` | string | 认证描述 |
| `auth_addr` | string | 认证地址 |
| `template_check_status` | int | **审核状态**（列表展示用） |
| `is_top` | int | 是否置顶 |

### 5.2 publish_status 枚举值

| 值 | 含义 |
|----|------|
| **1** | 未审核、被管理侧拒绝、撤销（待重新提交） |
| **2** | 审核中、审核通过 |

> ⚠️ `publish_status = 1` 时详情页右上角**不显示**「发布」按钮；`publish_status = 2` 时**显示**「发布」按钮。

### 5.3 template_check_status 枚举值

| 值 | 状态 | 说明 |
|----|------|------|
| `-1` | 未审核 | 刚创建，尚未提交审核 |
| `1` | 待审核 | 已提交，等待管理侧审核 |
| `2` | 通过 | 审核通过 |
| `3` | 拒绝 | 审核不通过 |
| `4` | 撤销 | 用户主动撤销 |

### 5.4 详情页

**基本信息接口**：`GET /api/mcp/detail`（MCP）、`GET /api/skill/detail`（Skill）

**工具列表接口**：`GET /api/mcp/tools/list`（MCP）、`GET /api/skill/tools/list`（Skill）

### 5.5 详情页「发布」按钮显示规则

```
if (publish_status === 2) {
  显示「发布」按钮
} else {
  不显示「发布」按钮
}
```

## 六、publish_status 与 template_check_status 对照

| template_check_status | 含义 | publish_status 值 |
|------------------------|------|--------------------|
| `-1` 未审核 | 刚创建，未提交审核 | = 1 |
| `1` 待审核 | 已提交，等待审核 | = 2 |
| `2` 通过 | 审核通过，已上线 | = 2 |
| `3` 拒绝 | 审核不通过 | = 1 |
| `4` 撤销 | 用户主动撤销 | = 1 |

           │                                     │
           │                                     │
           ▼                                     ▼
┌─────────────────────┐           ┌─────────────────────┐
│   管理侧「下架」      │           │  用户点击发布按钮     │
│  template_check 维持  │           │  重新走发布流程       │

│  详情页无发布按钮     │

           │
           │ 用户点击发布按钮
           ▼
    重新走发布流程
