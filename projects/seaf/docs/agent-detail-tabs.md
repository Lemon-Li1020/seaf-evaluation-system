## 日志模块

### 一、列表展示

日志列表直接显示接口返回的调用日志，包含以下字段：

| 字段 | 说明 |
|------|------|
| **用户名称** | 调用智能体的用户名，取值：`t_user.user_name` |
| **会话 ID** | 当前对话的会话标识 |
| **日志 ID** | 唯一日志标识 |
| **对话问题** | 用户提问/触发内容 |
| **操作** | 两个入口：日志 / 执行链（见下方） |

### 二、「日志」操作

点击「**日志**」→ 打开**日志回放**

展示内容：本次对话的完整记录（用户提问 → 智能体响应 → 中间过程）

### 三、「执行链」操作

点击「**执行链**」→ 打开智能体的 **Langfuse** 链接

展示内容：智能体整个调用链路（从发起 → 各环节处理 → 最终输出）

### 四、用户名称取值说明

> ⚠️ 以下调用方式可能导致用户名称为空，后续需补充处理逻辑

| 调用方式 | 用户名称 |
|----------|----------|
| API 调用 | 可能为空 |
| 分享链接调用 | 可能为空 |
| 用户侧调用 | 可能为空 |
