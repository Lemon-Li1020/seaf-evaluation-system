## 一、背景与目标

### 1.1 背景

随着大语言模型（LLM）应用的快速普及，企业级 AI 智能体（Agent）正在成为主流交互形态。然而，智能体的质量保障面临严峻挑战：

| 现状痛点 | 描述 |
|----------|------|
| **评测效率低** | 依赖人工逐条打分，评测集规模受限，无法支撑高频迭代 |
| **质量回归风险** | 每次模型升级或提示词变更，无法量化效果变化，导致上线后质量滑坡 |
| **评测维度单一** | 仅关注最终回答质量，无法区分工具调用、知识库召回、多 Agent 协作等不同层次的能力 |
| **智能化程度不足** | 缺乏置信度机制、自动化触发和版本对比能力 |

### 1.2 目标

**核心目标**：构建一套**通用、可扩展、自动化**的智能体评测系统，实现：

1. **自动打分** — 去掉 80%+ 人工打分工作量
2. **版本回归** — 量化每次变更的效果变化
3. **多维评测** — 支持不同类型智能体的差异化评测
4. **无人值守** — 接入 CI/CD，发布前自动评测，不合格阻断上线

## 二、智能体分类体系

本系统支持的智能体类型及评测重点：

### 2.1 智能体类型定义

| 类型 | 核心机制 | 用户价值 | 典型场景 |
|------|----------|----------|----------|
| **推理智能体** | 单 Agent + 多工具协作（知识库/Skill/MCP/模型） | 精准任务执行 | 客服问答、数据查询、内容生成 |
| **编排智能体** | 多 Agent 协作编排 | 复杂任务分解 | 竞品分析、报告生成、多步骤工作流 |
| **工作流智能体** | DAG 节点串联 | 流程自动化 | 审批流、数据处理流水线 |

### 2.2 各类型核心评测维度

#### 推理智能体

| 评测维度 | 权重 | 说明 |
|----------|------|------|
| 意图识别准确率 | 20% | 是否正确理解用户意图 |
| 工具调用准确性 | 25% | 是否调用正确的工具，参数是否正确 |
| 知识库召回质量 | 20% | 检索内容是否相关、完整 |
| 答案准确性 | 20% | 最终回答是否正确解决了问题 |
| 答案完整性 | 10% | 是否回答了用户问题的全部子问题 |
| 格式规范性 | 5% | 输出格式是否符合预期 |

#### 编排智能体

| 评测维度 | 权重 | 说明 |
|----------|------|------|
| 编排合理性 | 30% | 是否调用了正确的子 Agent，顺序是否合理 |
| 结果聚合质量 | 25% | 是否正确合并多 Agent 输出，无冲突/遗漏 |
| 死循环检测 | 15% | 是否存在异常重复调用 |
| 端到端效果 | 20% | 最终回答是否解决了用户问题 |
| 工具调用准确性 | 10% | 子 Agent 的工具调用是否正确 |

#### 工作流智能体

| 评测维度 | 权重 | 说明 |
|----------|------|------|
| 流程完整性 | 25% | DAG 是否跑完整，核心节点是否全部执行 |
| 节点失败处理 | 25% | 失败后是否有正确的降级/重试/终止策略 |
| 响应时间合理性 | 25% | 各节点耗时是否在合理范围内 |
| 端到端耗时 | 15% | 从启动到结果返回的总时长 |
| 结果质量 | 10% | 最终输出是否正确 |

## 三、功能需求

### 3.1 评测集管理

| 功能 | 描述 | 优先级 |
|------|------|--------|
| **评测集 CRUD** | 按智能体类型管理评测集，支持创建/编辑/删除/查询 | P0 |
| **批量导入/导出** | 支持 JSON/CSV 格式批量导入评测数据 | P0 |
| **评测集版本管理** | 评测集变更记录，支持版本对比和回滚 | P1 |
| **评测集分层** | 分为基础集（必测）、扩展集（可选）、回归集（发布前），支持按场景打标签 | P1 |

**评测数据结构：**

```json
{
  "id": "唯一标识",
  "name": "评测项名称",
  "agent_type": "reasoning | orchestration | workflow",
  "query": "用户问题/触发条件",
  "expected_tools": ["期望调用的工具列表"],
  "expected_answer_pattern": "期望答案正则或关键词（可选）",
  "evaluation_dimensions": ["accuracy", "tool_usage", "completeness"],
  "performance_baseline": {
    "p95_latency_ms": 5000,
    "max_latency_ms": 10000
  },
  "tags": ["标签1", "标签2"],
  "difficulty": "easy | medium | hard",
  "version": "v1.0"
}
```

### 3.2 评测任务执行

| 功能 | 描述 | 优先级 |
|------|------|--------|
| **手动触发评测** | 选择评测集 + 智能体，执行评测任务，实时显示进度 | P0 |
| **评测进度查看** | 支持查看当前评测任务的进度、耗时、已完成数量 | P0 |
| **并发评测** | 支持同时运行多个评测任务，互不干扰 | P1 |
| **评测中断/恢复** | 支持中断正在运行的评测任务，并从断点恢复 | P2 |

### 3.3 LLM 自动打分（核心）

| 功能 | 描述 | 优先级 |
|------|------|--------|
| **多维打分** | 接入 LLM 评委，对每个评测项按维度独立打分 | P0 |
| **加权总分** | 根据预设权重计算加权总分（满分 100） | P0 |
| **置信度判断** | LLM 评委输出置信度（高/中/低），决定是否人工复核 | P0 |
| **评分稳定性** | 同一评测项连续跑 3 次取中位数，标准差 > 15 标记不稳定 | P1 |
| **评委模型选择** | 支持配置不同 LLM 作为评委（推荐 DeepSeek-V3） | P0 |
| **评委 Prompt 模板** | 支持按智能体类型配置不同 Prompt 模板 | P0 |

**打分流程：**

```
用户发起评测
    ↓
查询评测集，逐条执行
    ↓
Agent 返回响应 + 工具调用日志 + 耗时
    ↓
调用 LLM 评委打分（多维评分 + 置信度）
    ↓
置信度高(>90%) → 直接采纳
置信度中(70-90%) → 采纳，标记关注
置信度低(<70%) → needs_human_review=true，推送人工复核队列
    ↓
汇总生成评测报告
```

### 3.4 版本回归对比

| 功能 | 描述 | 优先级 |
|------|------|--------|
| **版本对比视图** | 新版 vs 旧版多维打分对比，差异高亮显示 | P0 |
| **回归检测告警** | 分数下降超过阈值（如 10%）→ 阻断发布 + 通知 | P0 |
| **趋势图表** | 各版本分数趋势折线图，支持按维度筛选 | P1 |
| **历史版本管理** | 保存历史评测记录，支持查询和对比任意版本 | P0 |

### 3.5 自动触发机制

| 功能 | 描述 | 优先级 |
|------|------|--------|
| **发布前钩子** | 智能体发布前自动触发评测，不合格阻断发布流程 | P1 |
| **定时评测** | 支持配置每日/每周定时执行全量或指定评测集 | P2 |
| **CI/CD 集成** | 提供 API 和 Webhook，支持接入流水线 | P1 |

### 3.6 结果展示与导出

| 功能 | 描述 | 优先级 |
|------|------|--------|
| **评测报告总览** | 通过率、总分、分项得分、Grade（ABCDF）、响应耗时 | P0 |
| **分类型报告** | 按智能体类型生成独立报告 | P1 |
| **慢节点定位** | 工作流评测中耗时超阈值节点 TOP N | P1 |
| **人工复核队列** | 置信度低或评分不稳定的评测结果，支持在线打分 | P2 |
| **报告导出** | 支持导出 PDF/Excel/JSON 格式报告 | P1 |

## 四、非功能需求

### 4.1 性能指标

| 指标 | 要求 |
|------|------|
| 单次评测任务耗时 | ≤ 5min（按 50 题计算，含 LLM 评委调用） |
| 并发能力 | 支持至少 5 个评测任务同时运行 |
| LLM 评委调用成功率 | ≥ 99%，失败自动重试 3 次 |
| 评测结果查询响应 | ≤ 1s（历史记录查询） |

### 4.2 可扩展性

| 需求 | 说明 |
|------|------|
| 智能体类型扩展 | 支持新增智能体类型，只需配置新的评测维度和 Prompt 模板 |
| 评委模型扩展 | 支持接入新的 LLM 作为评委，按需切换 |
| 评测维度扩展 | 支持在已有智能体类型下新增评测维度 |
| 平台集成扩展 | 提供标准化 API，支持与第三方平台对接 |

### 4.3 安全与隔离

| 需求 | 说明 |
|------|------|
| 数据隔离 | 不同租户/团队的评测数据完全隔离 |
| 权限控制 | 支持按角色分配权限（管理员/评测员/查看者） |
| 敏感信息处理 | 评测数据中的敏感信息自动脱敏 |
| 审计日志 | 所有操作留痕，支持追溯 |

## 五、Grade 评级体系

| Grade | 分数区间 | 含义 | 行动 |
|--------|----------|------|------|
| **A** | 90-100 | 优秀，可上线 | 直接发布 |
| **B** | 80-89 | 良好，可上线 | 建议优化后发布 |
| **C** | 70-79 | 合格，需关注 | 建议修复低分项后发布 |
| **D** | 60-69 | 不合格，禁止上线 | 必须修复后重新评测 |
| **F** | < 60 | 严重不合格 | 阻断发布，需紧急修复 |

**Grade 计算规则：**
- 加权总分 ≥ 90 → A
- 加权总分 80-89 → B
- 加权总分 70-79 → C
- 加权总分 60-69 → D
- 加权总分 < 60 → F
- 任一维度得分 < 50 → Grade 降一级
- 任一维度得分 < 40 → Grade 再降一级

## 六、LLM 评委 Prompt 设计规范

### 6.1 Prompt 结构

每个类型的 Prompt 遵循统一结构：

```
【角色定义】你是一个专业的 AI 评测专家
【输入信息】用户问题 / Agent 回答 / 工具调用日志 / 期望答案
【评分维度】每个维度独立打分 0-100，说明评分标准
【置信度判断】判断自己评分的置信度（高/中/低）
【输出格式】JSON 结构化结果
```

### 6.2 标准输出格式

```json
{
  "scores": {
    "维度A": 85,
    "维度B": 90
  },
  "weighted_score": 87.5,
  "confidence": "high | medium | low",
  "needs_human_review": false,
  "key_findings": "核心发现说明",
  "main_issues": ["问题1", "问题2"],
  "suggestions": ["建议1", "建议2"]
}
```

### 6.3 置信度阈值

| 置信度 | 阈值 | 处理方式 |
|--------|------|----------|
| 高置信 | > 90% | 直接采纳，无需复核 |
| 中置信 | 70-90% | 采纳，标记关注 |
| 低置信 | < 70% | needs_human_review=true，推送人工复核 |

## 九、风险与应对

| 风险 | 可能性 | 影响 | 应对措施 |
|------|--------|------|----------|
| LLM 评委打分不稳定 | 高 | 中 | 多次采样取中位数，设置置信度兜底 + 人工复核 |
| 评测集建设初期成本高 | 高 | 中 | 优先覆盖高频场景（P0 场景），逐步扩充 |
| LLM API 调用成本高 | 中 | 低 | 优先使用低成本模型（DeepSeek-V3），按需升级 |
| 依赖 LLM API 稳定性 | 中 | 中 | 配置降级策略，API 失败自动切换备选模型 |
| 评测结果与用户体验差异 | 中 | 中 | 定期收集用户反馈，持续优化评测集和评分标准 |

## 十一、术语表

| 术语 | 定义 |
|------|------|
| **智能体（Agent）** | 基于 LLM 的具备自主任务执行能力的 AI 系统 |
| **评测集（Test Set）** | 一组标准化的测试用例，用于评估智能体能力 |
| **LLM 评委** | 使用 LLM 自动评估 Agent 回答质量的技术 |
| **意图识别** | 判断用户问题背后的真实目的和需求 |
| **工具调用** | Agent 调用外部工具（API/MCP/Skill）完成特定任务 |
| **编排（Orchestration）** | 多 Agent 协作完成任务的方式 |
| **工作流（Workflow）** | DAG 节点串联的流程执行方式 |
| **置信度** | LLM 评委对自身评分结果的确定程度 |
| **回归检测** | 检测新版相对于旧版是否有质量下降 |
| **Grade** | 综合评级，分为 A/B/C/D/F 五档 |
