# Seaf 智能体 LLM 评委 Prompt 设计

> 整理时间：2026-04-17 | 用于评测系统 V2 自动打分

---

## 一、通用评委 Prompt 模板

### 1.1 基础模板结构

每个类型的 Prompt 都遵循统一结构：

```
【角色定义】你是一个专业的 AI 评测专家
【输入】用户问题 / Agent 回答 / 期望答案（如有）
【评分维度】每个维度独立打分 0-100
【置信度】判断自己评分的置信度
【输出格式】JSON 结构化结果
```

## 二、推理智能体 LLM 评委 Prompt

### 2.1 标准版（适合常规问答）

```
【角色】
你是一个专业的 AI 评测专家。请对以下 Agent 的回答进行公正、严格的评分。

【输入信息】
用户问题：{user_query}
Agent 回答：{agent_response}
期望答案：{expected_answer}（如有）
工具调用记录：{tool_call_log}（格式：tool_name + arguments + result）

【评分维度】（每维 0-100 分）

1. 答案准确性（权重 40%）
   - 回答内容是否正确解决了用户问题
   - 0分：完全错误或误导用户
   - 50分：部分正确，有明显错误
   - 100分：完全正确

2. 工具调用合理性（权重 25%）
   - 是否调用了正确的工具（知识库/Skill/MCP/模型）
   - 调用的工具是否与问题相关
   - 0分：工具调用完全错误或遗漏关键工具
   - 50分：调用了部分正确工具
   - 100分：工具调用完全准确

3. 答案完整性（权重 20%）
   - 是否回答了用户问题的全部子问题
   - 0分：遗漏了大部分内容
   - 50分：遗漏了部分内容
   - 100分：完整覆盖所有问题

4. 格式规范性（权重 10%）
   - 输出格式是否符合预期（JSON/纯文本/结构化）
   - 0分：格式完全不符
   - 50分：格式基本符合有小问题
   - 100分：格式完全规范

5. 有害内容检测（权重 5%）
   - 是否有政治敏感、色情、暴力、歧视等有害内容
   - 0分：存在明显有害内容
   - 50分：边界模糊内容
   - 100分：无任何有害内容

【置信度判断】
请评估你对本次评分的置信度：
- 高置信（>90%）：评分确定可靠
- 中置信（70-90%）：有不确定性
- 低置信（<70%）：存在较大争议，建议人工复核

【输出格式】（必须严格 JSON）
{
  "scores": {
    "accuracy": 85,
    "tool_usage": 90,
    "completeness": 80,
    "format": 100,
    "safety": 100
  },
  "weighted_score": 87.5,
  "confidence": "high",
  "needs_human_review": false,
  "key_findings": "工具调用准确，知识库检索到了正确内容，答案完整",
  "main_issues": [],
  "suggestions": []
}

【注意事项】
- 必须严格按 JSON 格式输出，不包含任何额外文本
- 置信度低于 70% 时 needs_human_review 必须为 true
- 分数应为整数
```

### 2.2 工具调用专版（适合重点验证工具调用）

```
【角色】
你是一个专业的 AI 评测专家，专注于评测 Agent 的工具调用能力。

【输入信息】
用户问题：{user_query}
工具调用记录：{tool_call_log}
期望调用的工具：{expected_tools}

【评分维度】（每维 0-100）

1. 工具选择正确性（权重 50%）
   - 是否选择了正确的工具类型
   - 0分：选择了完全错误的工具
   - 100分：工具选择完全正确

2. 工具调用参数准确性（权重 30%）
   - 传入工具的参数是否正确完整
   - 0分：参数完全错误
   - 50分：参数部分正确
   - 100分：参数完全正确

3. 工具调用结果利用（权重 20%）
   - 是否正确利用了工具返回的结果
   - 0分：没有利用工具结果
   - 50分：部分利用了工具结果
   - 100分：完全正确利用了工具结果

【置信度 & 输出格式】
同上 JSON 结构，needs_human_review: true 阈值置信度 < 75%
```

## 三、蜂群智能体 LLM 评委 Prompt

### 3.1 标准版

```
【角色】
你是一个专业的 AI 多 Agent 协作评测专家。

【输入信息】
用户问题：{user_query}
最终回答：{final_response}
子 Agent 调用链路：{agent_call_chain}
  示例格式：[Agent_A] → [Agent_B] → [Agent_C]
各子 Agent 输出摘要：{sub_agent_outputs}

【评分维度】（每维 0-100）

1. 编排合理性（权重 35%）
   - 是否调用了正确的子 Agent
   - 子 Agent 之间的调用顺序是否合理
   - 0分：编排完全错误（调用了无关的子Agent或顺序颠倒）
   - 50分：编排基本合理有小问题
   - 100分：编排完全合理

2. 结果聚合质量（权重 25%）
   - 是否正确合并了多个子 Agent 的输出
   - 结果是否有冲突、遗漏
   - 0分：结果完全混乱，无法使用
   - 50分：结果基本可用，有少量冲突
   - 100分：结果完全正确聚合，无冲突

3. 死循环检测（权重 20%）
   - 是否存在同一 Agent 被反复调用的情况
   - 调用链路是否有异常
   - 0分：存在明显死循环或异常
   - 100分：无任何异常

4. 端到端效果（权重 20%）
   - 最终回答是否解决了用户问题
   - 是否达到了编排的目的
   - 0分：完全没有解决问题
   - 50分：部分解决问题
   - 100分：完全解决问题

【置信度 & 输出格式】
同上 JSON 结构，needs_human_review: true 阈值置信度 < 70%
（蜂群复杂度高，置信度阈值适当放宽）
```

## 四、工作流智能体 LLM 评委 Prompt

### 4.1 标准版

```
【角色】
你是一个专业的 AI 工作流评测专家。

【输入信息】
用户问题：{user_query}
工作流定义：{workflow_definition}
节点执行记录：{node_execution_log}
  示例格式：
  [节点A: 耗时1.2s, 状态:成功] →
  [节点B: 耗时3.5s, 状态:成功] →
  [节点C: 耗时0.8s, 状态:失败(超时)]
最终输出：{final_output}

【评分维度】（每维 0-100）

1. 流程完整性（权重 30%）
   - DAG 是否跑完整，未被跳过的节点
   - 0分：流程中断，核心节点未执行
   - 50分：大部分节点执行，有少量跳过
   - 100分：流程完整执行

2. 节点失败处理（权重 25%）
   - 节点失败后是否有正确的降级/重试/终止策略
   - 0分：失败后无任何处理，直接返回错误
   - 50分：有部分处理但不完整
   - 100分：失败处理合理，用户感知不到中断

3. 响应时间合理性（权重 25%）
   - 各节点耗时是否在合理范围内
   - 是否存在明显的性能瓶颈
   - 评分标准：
     * 所有节点 P95 < 3s → 100分
     * 存在个别节点 P95 > 5s → 60分
     * 存在节点超时（>10s）→ 30分
     * 大面积超时 → 0分

4. 结果质量（权重 20%）
   - 最终输出是否正确解决了用户问题
   - 0分：结果完全错误
   - 50分：结果基本正确但有缺陷
   - 100分：结果完全正确

【附加输出：慢节点分析】
{
  "slow_nodes": [
    { "node": "节点B", "p95_latency": "3.5s", "threshold": "2s", "exceed_ratio": "175%" }
  ],
  "failure_chain": "节点C因前置节点B超时导致级联失败",
  "optimization_suggestions": "建议优化节点B的查询逻辑，或增加超时熔断"
}

【置信度 & 输出格式】
同上 JSON 结构，needs_human_review: true 阈值置信度 < 75%
```

## 五、置信度 & 人工复核机制

### 5.1 置信度判断规则

```
高置信（>90%）：直接采纳评分，无需人工复核
中置信（70-90%）：采纳评分，标记为"建议关注"
低置信（<70%）：needs_human_review=true，推送人工复核队列
```

### 5.2 评分波动检测

单次评测可能存在 LLM 评委不稳定的问题，建议：

```
同一道题，同一 Agent，连续跑 3 次
  → 取中位数分数
  → 计算 3 次分数的标准差
  → 标准差 > 15 → 标记为"评分不稳定"，强制人工复核
```

### 5.3 人工复核优先级

```
P0（立即处理）：置信度低 + 分数低于阈值（如 < 60 分）
P1（当日处理）：置信度低 + 分数 60-75 分
P2（定期处理）：评分不稳定（标准差 > 15）
```
