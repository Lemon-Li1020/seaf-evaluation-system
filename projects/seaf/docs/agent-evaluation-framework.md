# Seaf 智能体评测框架

> 整理时间：2026-04-17 | 整理人：根据李文华口述

---

## 一、智能体分类

Seaf 平台智能体分为三种类型：

| 类型 | 核心机制 | 用户关注点 |
|------|----------|-----------|
| **推理智能体** | 单 Agent，多工具协作（提示词 + 知识库 + Skill + MCP + 模型） | 工具是否按用户意图调用，多轮对话是否正常 |
| **蜂群智能体** | 编排多个推理智能体协作 | 是否正确调用编排的子 Agent，编排逻辑是否合理 |
| **工作流智能体** | DAG 节点串联，顺序/并行执行 | 各节点响应时间，整体流程完整性 |

---

## 二、各类型评测重点

### 2.1 推理智能体

**本质**：单 Agent，多工具协作

| 评测点 | 评测方法 |
|--------|----------|
| **意图识别** | 给定用户问题，校验是否调用了正确的工具 |
| **工具调用准确率** | Mock 工具调用，校验 tool_name + arguments |
| **参数填充** | 校验传入工具的参数是否完整正确 |
| **多轮对话保持** | 注入上下文，验证关键信息在各轮次中被正确使用 |
| **知识库召回** | 给定问题，校验是否检索到相关内容 |
| **幻觉检测** | LLM 评委检测回答是否超出知识库范围 |

**评测集示例**：

```
题目：用户问"我们的退换货政策是什么"
期望：调用知识库检索，退换货相关内容
错误：调用了 MCP 天气接口

题目：用户连续问了3个关于合同的问题
期望：第3轮仍记得第1轮合同内容
错误：第3轮丢失上下文
```

### 2.2 蜂群智能体

**本质**：编排多个推理智能体协作

| 评测点 | 评测方法 |
|--------|----------|
| **子 Agent 调用** | 记录是否调用了正确的子 Agent |
| **调用顺序** | 校验编排顺序是否合理 |
| **结果聚合** | 多子 Agent 结果是否正确合并/去重 |
| **死循环检测** | 同一子 Agent 被调用次数是否超限 |
| **冲突处理** | 多 Agent 返回不一致时如何处理 |
| **端到端效果** | 最终输出是否满足用户原始意图 |

**评测集示例**：

```
场景：用户问"帮我分析这份竞品报告，并给出建议"
期望：文档解析 Agent → 数据分析 Agent → 报告生成 Agent
错误：跳过了数据分析直接生成报告

场景：两个子 Agent 给出了相反的结论
期望：蜂群层有冲突检测和结论取舍
错误：直接输出两个冲突结论
```

### 2.3 工作流智能体

**本质**：DAG 节点串联

| 评测点 | 评测方法 |
|--------|----------|
| **节点响应时间** | 每个节点的耗时 P50/P95/P99 |
| **流程完整性** | DAG 是否跑完整，中间节点是否跳过 |
| **节点失败处理** | 单节点失败后是否正确降级/重试/终止 |
| **并发能力** | 可并行的节点是否真正并行执行 |
| **端到端耗时** | 从启动到结果返回的总时长 |
| **资源消耗** | Token 消耗、内存峰值 |

**核心指标**：

```
整体流程成功率 = 成功完成的流程数 / 总流程数
节点 P95 耗时 = 第 95 百分位节点响应时间
慢节点定位 = 耗时超过阈值（如 5s）的节点 TOP N
```

## 六、评测系统升级路线图

### 当前 V1（已有）
- 批量创建评测集（query）
- 手动执行评测任务
- 返回 Agent 最终响应 + 平均响应时间
- 人工逐条打分

### V2 目标：自动打分
**核心：去掉人工打分，接入 LLM 评委**

| 功能 | V1 | V2 |
|------|-----|-----|
| 评测集管理 | query | 增加评分维度字段 |
| 自动打分 | ❌ | ✅ LLM 评委 |
| 评测结果 | 响应 + 时间 | 增加多维打分 + 版本对比 |
| 人工复核 | 全量 | 仅置信度低（<70%） |

**V2 落地步骤：**
1. 评测集增加维度字段（准确性/工具调用/完整性/格式/有害内容）
2. 接入 LLM 评委（推荐 DeepSeek-V3，性价比最高）
3. 置信度 < 70% 触发人工复核
4. 同一题跑 3 次取中位数，标准差 > 15 强制人工复核

**V2 预计工时：2 周**

### V3 目标：无人值守 + 版本回归
**核心：定时触发，版本 diff，告警通知**

| 功能 | 说明 |
|------|------|
| 自动触发 | 智能体发布前自动跑评测，不合格阻断 |
| 版本 diff | 新版 vs 旧版，分数对比，回归检测 |
| 告警通知 | 分数下降超阈值 → 推送通知 |
| 评测集分层 | 基础集/扩展集/回归集分离 |

**V3 预计工时：2-3 周**

### 各阶段价值

| 阶段 | 去掉人工工作量 | 评测频率 | 发布质量 |
|------|--------------|----------|----------|
| V1 当前 | 0% | 低（手动） | 依赖人工 |
| V2 | ~80% | 中（手动） | 自动 + 人工兜底 |
| V3 | ~95% | 高（自动） | 无人值守 |
